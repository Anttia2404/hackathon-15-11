MACHINE LEARNING - BÀI GIẢNG CƠ BẢN

1. GIỚI THIỆU
Machine Learning là nhánh của trí tuệ nhân tạo cho phép máy tính học từ dữ liệu mà không cần lập trình cụ thể từng bước. Thay vì viết code cho từng trường hợp, chúng ta cung cấp dữ liệu và để máy tính tự học patterns.

2. CÁC LOẠI MACHINE LEARNING

2.1. Supervised Learning (Học có giám sát)
- Sử dụng dữ liệu có nhãn (labeled data)
- Model học mối quan hệ giữa input và output
- Ví dụ: Phân loại email spam, dự đoán giá nhà

2.2. Unsupervised Learning (Học không giám sát)
- Sử dụng dữ liệu không có nhãn
- Model tự tìm patterns và cấu trúc trong dữ liệu
- Ví dụ: Phân nhóm khách hàng, giảm chiều dữ liệu

2.3. Reinforcement Learning (Học tăng cường)
- Model học qua thử và sai
- Nhận reward khi làm đúng, penalty khi làm sai
- Ví dụ: Game AI, robot tự động

3. NEURAL NETWORK

Neural Network mô phỏng cách hoạt động của não người với các lớp neurons kết nối với nhau. Mỗi neuron nhận input, xử lý và truyền output sang neuron tiếp theo.

Deep Learning là một nhánh của ML sử dụng neural network với nhiều lớp ẩn (hidden layers) để học các đặc trưng phức tạp từ dữ liệu.

4. VẤN ĐỀ OVERFITTING

Overfitting xảy ra khi model học quá chi tiết từ training data, bao gồm cả nhiễu và các đặc điểm không quan trọng. Điều này dẫn đến model hoạt động rất tốt trên training data nhưng kém trên test data.

Cách khắc phục:
- Regularization: Thêm penalty vào loss function
- Cross-validation: Chia dữ liệu thành nhiều fold
- Early stopping: Dừng training khi validation loss tăng
- Dropout: Tắt ngẫu nhiên một số neurons trong training

5. FEATURE ENGINEERING

Feature engineering là quá trình tạo ra các features mới từ dữ liệu thô để cải thiện hiệu suất model. Đây là bước QUAN TRỌNG NHẤT trong ML, thậm chí quan trọng hơn cả việc chọn thuật toán.

Các kỹ thuật:
- Feature scaling: Chuẩn hóa dữ liệu
- Feature selection: Chọn features quan trọng
- Feature creation: Tạo features mới từ features cũ
- Handling missing values: Xử lý dữ liệu thiếu

6. ĐÁNH GIÁ MODEL

Các metrics phổ biến:
- Accuracy: Tỷ lệ dự đoán đúng
- Precision: Độ chính xác của positive predictions
- Recall: Tỷ lệ tìm được positive cases
- F1-score: Trung bình điều hòa của Precision và Recall
- ROC-AUC: Đánh giá khả năng phân loại

7. KẾT LUẬN

Machine Learning là công cụ mạnh mẽ để giải quyết các bài toán phức tạp. Thành công trong ML phụ thuộc vào:
1. Dữ liệu chất lượng
2. Feature engineering tốt
3. Chọn model phù hợp
4. Tránh overfitting
5. Đánh giá đúng đắn
